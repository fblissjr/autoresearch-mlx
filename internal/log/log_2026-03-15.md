# Session Log: 2026-03-15

## Upstream sync from karpathy/autoresearch

### What changed
Compared our MLX port against the upstream karpathy/autoresearch codebase (local reference at `coderef/autoresearch-karpathy`) and synced 4 algorithmic improvements:

1. **`estimate_flops` method** -- upstream properly excludes embedding/scalar params from base FLOPs and adds per-window attention FLOPs. Our naive `6 * num_params` was overestimating by ~32%.

2. **Muon momentum schedule** -- upstream ramps momentum from 0.85 to 0.95 over 300 steps. We had it fixed at 0.95 from step 0. Added via `_schedulers['momentum']` on the Muon optimizer.

3. **Weight decay schedule** -- upstream linearly decays weight decay to 0 over training. We had constant WD. Added via `_schedulers['weight_decay']` on the Muon optimizer.

4. **NaN check** -- upstream checks `math.isnan(loss) or loss > 100`. We only had `> 100`.

- CUDA-specific: FA3, torch.compile, pin_memory, device transfers

### Key technical finding
The `estimate_flops` method reveals our old FLOPs estimate was inflated by ~32% because it counted all parameters (including embeddings and scalars) in the `6*N` formula. The new estimate correctly only counts matrix FLOPs from transformer layers + lm_head, plus attention FLOPs per window size.
